## 🤖 Claude · 2026-06-30 13:43 UTC

Autoformatting on save in a C# project usually comes from one of two places: **VS Code's format-on-save setting** or an **`.editorconfig`** that the C# extension applies. Here's how to turn it off.

## 1. VS Code setting (most common)

Add this to your workspace `.vscode/settings.json` (or your user settings):

```json
{
  "editor.formatOnSave": false
}
```

To disable it **only for C#** (keep it for other languages):

```json
{
  "[csharp]": {
    "editor.formatOnSave": false
  }
}
```

## 2. Other on-save actions

The C# Dev Kit / OmniSharp can also run fixes and import organizing on save. Disable those too if present:

```json
{
  "editor.codeActionsOnSave": {
    "source.fixAll": "never",
    "source.organizeImports": "never"
  },
  "omnisharp.enableEditorConfigSupport": false
}
```

## 3. `.editorconfig`

If there's an `.editorconfig` at your repo root, formatting rules in it are applied by the formatter. Disabling `formatOnSave` (step 1) stops it from running automatically; you don't need to delete the file.
